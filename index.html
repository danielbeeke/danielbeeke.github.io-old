Fundaments

Company Fundaments
My role System engineer / architect
Period 2018 / 2020
System VueJs, Composer, CodeIgniter, Doctrine, ExactOnline and a lot of integrations

The hosting company Oxilion needed a rebuild of their backbone system for customers and engineers. It was old and a lot of new business choices made a rebuild necessary. While working on this project, Oxilion merged with their sister company. This resulted in one big multi-tenant application, serving both the customers and engineers of Oxilion and Fundaments.

Changing situations had made the old system inconsistent, one of the challenges of the new system is to get some of the data more consistent. Oxilion sells from ssl certificates to hosting, VPS’es and managed hosting to private cloud solutions.

Fundaments sells more to enterprise: virtual data center provided by vCloud and enterprise backup solutions from Zerto and Cohesity.

Let’s start with a little but important hosting component, the DNS panel. Here you see the screen to manage your DNS records.

It has nice features such as restoring them from a previous time, or copying them over from another domain. This form is built with Vue. When saved, it sends them back to PowerDNS, the DNS backend of Oxilion.

All the records are validated while you type. It is also available on mobile.








The system has a lot of integrations. The biggest integration is Exact Online. Exact Online provides the products, the subscriptions, the invoices, the customers and a lot more. OpenProvider is used for SSL certificates, the process of buying and renewing is fully automated. DirectAdmin and Plesk are used for shared hosting. A customer can order it via the website. DNS is managed with PowerDNS.








Oxilion provides an anti-spam solution for all the purchased domains and it also provides it in a subscription form. Monitoring is done with Zabbix, it is integrated so the engineers can process failures of servers. For server backups Cohesity is used. Duo provides two-factor authentication.

One of the things Oxilion sells is webhosting. That can be on Plesk or DirectAdmin, you can directly login to your control panel.

The system is tightly integrated with ExactOnline. Customers, products, subscriptions and a lot more things are stored inside ExactOnline. Customers can have multiple users and users can have multiple customers. If a user has multiple customers a customer switcher will appear.

To switch from the old version of the backbone to the new one we needed to import all the customers and subscriptions. This was done with an importer.

In the most left column you see a couple of sources, this is a screenshot of a development version. These sources are validated automatically. When some rows were not valid for the new system it would be detected and someone would manually fix it in this import wizard. THe migrations are all finished, and it worked successfully.

Kozijnisolatie

Company & client While at Fonkel with Graphius for NBvT
Languages JS
Systems VueJs, Babel, JSPM, Drupal

The dutch branch organization for carpentry factories, NBvT has a lot of members that need to do a lot of difficult calculations to show how much warmth gets lost in their wooden constructions. They need to tell their customers how much warmth their wooden doors, frames and windows let through.

This tool helps them by supplying a user interface in which they can build the whole frame digitally and set all the right parameters. With that input the tool calculates how much warmth the customer will lose.

The tool needs to know a few basic settings, like the wood type that will be used and the sizes of the frame. This screen we built together with the technical product owner, he explained a lot of things that we needed to learn about windows and doors.


Some technical information: this whole tool runs inside the browser of the user. It is super fast because of that. The tools saves the data in the NBvT’s website. It does that in a fully decoupled way. A project is exported to a text definition in JSON format and that is saved in the user’s account.

One of the difficult things in this project was building things that we have never build before. You need to have a model that matches the reality or at least a simplification that works for the use case.

Creating the grid and the validation for it was interesting. Inserting rows after initiating the grid, removing columns, checking for incomplete squares, all things that needed small algorithms.

The next step involves changes the different lines of the grid so it has the right dimensions. After that we can continue to fill it up with doors, windows and wood panels.

All those lines where the sizes are shown are calculated and drawn with code.

All the images of doors and windows are drawn with code. Each part that is used in the calculation, like a metal strip or a piece of wood has dimensions and is drawable. For example, a window inside a door is a piece with dimensions.
A lot of things are adjustable, how thick the wood needs to be, the wood type, the aluminium type and a lot more!

Every piece of the wooden frame has a settings form. In that form you can tell how thick the wood or glass is. You have project defaults as shown right in the image above that you can override. The drawing is on scale. It is a SVG with the real sizes, but in points.

The last step shows the results. It does that with expandable details. The user can download a PDF to give to the customer.

In the last step it is also possible to check what would be different when you would use a different kind of wood.

Haringvliet

Company & client While at Fonkel with Graphius for WWF
Team & My role 3 developers, lead developer
Period 2017
Languages JS
Systems VueJs, Babel, JSPM, Drupal

A dozen of organisations from the Netherlands have bundled forces to advance and develop a region. This region has a lot of beautiful nature.

Our task was to create a news / blog website with an interactive map.

We did this in three iterations.

  1. We built the website with news, blogs and some project information.
  2. Then we built a first version of the map, with a map drawn by an illustrator.
  3. Then we made the map as shown in this showcase.

The project was built with Drupal, Leaflet and Mapbox Studio Classic. I created the map layer with CartoCss and created the initial architecture for the various components of the map.

This project was mostly made by me, it was nice, working on such a nicely designed project! It was designed by Graphius.

We had a different UI for mobile using our open sourced library ‘Sprinkhaan’. These three screens show the use of Sprinkhaan. It mimics the user experience of other map platforms. It was built using the new web animation API. More recently I converted Sprinkhaan to vanilla ES6 code.

Bike routes can be uploaded via .gpx files. You can switch on and off all kinds of layers. We have a specific layer for the road labels and also one for the bike signs.

Things I have learned from this project are: a little bit of Qgis and a lot of CartoCss. We build this while I was learning ES6. In this time I found a way to combine Drupal behaviours with transpiled ES6 code. I learned a lot of ES6 code and tricks while working on this project.

Hive

Project type Hobby project
Period 2018
Languages HTML, SCSS, ES6
Online View on github!

A friend introduced me to the game Hive. It is a strategic thinking game. It is a little bit similar to chess. This case will be a bit more technical explaining what I liked about building this game in javascript.

Technically this project is not finished, but it has enough things to talk about. It is built in ES6, the plan was to create a WebRTC multiplayer game just as the game Onitama that I digitalized.

In this project I learned a lot of ES6. The game is written in pure ES6, no Babel or some other transpiler. I have used Webcomponents to create the insects, the player stacks and the board. The game state is put into the HTML attributes.

The game is full of complex algorithms. The ones that I have yet to tackle are detecting multiple groups of insects and I have to create something that detects if a valid path is possible. I have read a lot of things about the A* algorithm and have tried to implement it. So far I have not yet succeeded.

While building this game I tried to generalise as much as possible. I have also tried to put all the specific rules into the right files. I like to think about the structure of a project and how to organize the project.

OpenGroup

Project type Side project
Period 2016, 2017, 2018
Systems VueJS, WebRTC
Languages HTML, SCSS, ES6
Online View it online!

What if WhatsApp was fully open? What if you could create a group to chat with people and have the data only stored on the participating phones / laptops? What if you could integrate almost anything to your group? Like chess or your company software etc. OpenGroup is a hobby project for which I would like to work on funding in the right time. I have already created a working proof of concept.

The full idea is a p2p communication framework with plugins on top of it. Think of a multi chat plugin, a chess game, video conferencing etc. It works with WebRTC.

A mockup of how OpenGroup could look. This mockup is basically like all the other chat apps. Like slack or Whatsapp etc. Left you see all the groups / people you are chatting with. In the right screen you see in the group menu chat, calendar, settings and info. This group menu can be expanded by plugins.

The start of the functioning proof of concept. So what does the proof of concept include? The application can be used with multiple persons in one group.

There are multiple ways to connect to each other. Firstly via a websockets server, secondly via text. The second option allows for the use of OpenGroup without servers. To be honest, if you are behind NAT you maybe need a TURN server.

A screen where I am chatting with myself. The idea is that anybody can write a plugin, consisting of hosted javascript and maybe some images and/or css. OpenGroup is built upon Vue and is very modular, a little bit like Drupal. For a developer it should be possible to integrate some thing very easy.

Storage for example, that is something the developer should not have to worry about, although it is decentralized and could come from multiple sources and have conflicting revisions, the developer should have an easy API.
